# Spec: `augx export` Command

## Purpose

Aggregate the rules content of all currently linked Augx modules into the native single-file (or directory) rule format expected by a target AI coding tool, so that the same module set powers Claude Code, Cursor, Windsurf, and GitHub Copilot in addition to Augment Code.

## Command Grammar

```
augx export --target <claude-code|cursor|windsurf|copilot|all>
            [--output <path>]
            [--dry-run]
            [--force]
            [--verbose]
```

### Flags

| Flag | Required | Default | Description |
|---|---|---|---|
| `--target` | Yes (unless `export.targets` in coordination.json) | none | One of the four tools or `all`. May appear once with a comma-separated value. |
| `--output` | No | per-target conventional path | Override output path. Only valid with a single `--target`. |
| `--dry-run` | No | false | Report resolved file list and projected byte count per target without writing. |
| `--force` | No | false | Overwrite even if drift is detected. |
| `--verbose` | No | false | Print symlink/copy fallback decisions and per-module sizes. |

If `--target` is omitted, `augx export` reads `.augment/coordination.json.export.targets`. If that is also absent, exit non-zero with a usage error.

## Per-Target Output Paths

| Target | Path | Format |
|---|---|---|
| `claude-code` | `CLAUDE.md` | Markdown |
| `cursor` | `.cursor/rules/augx.mdc` | Markdown + YAML frontmatter |
| `windsurf` | `.windsurfrules` | Markdown |
| `copilot` | `.github/copilot-instructions.md` | Markdown |
| `all` | all of the above | per format |

## Output Structure

Each target file is composed of:

1. **Banner** (always first, after any required frontmatter):
   ```
   <!-- GENERATED BY augx export - DO NOT EDIT -->
   <!-- augx-version: <semver> -->
   <!-- generated-at: <ISO-8601> -->
   <!-- source-hash: <16-hex> -->
   <!-- modules: <id>@<ver>, ... -->
   ```
2. **Optional MCP wiring snippet** (inlined only when a Beads MCP server is declared in `.vscode/mcp.json` or `.augment/mcp/servers.json`). Content comes from `augment-extensions/workflows/mcp/templates/beads.md`. The snippet describes how the consuming agent should query the Beads MCP server. No live task data is embedded.
3. **Per-module sections**, in alphabetical order of module ID:
   ```
   ## Module: <module-id> (v<version>)
   <rendered module rules content>
   ```
4. **Footer**:
   ```
   <!-- End of augx-generated content. Canonical modules: augment-extensions/modules/ -->
   ```

For `cursor`, the YAML frontmatter precedes the banner:

```yaml
---
description: Augx aggregated rules
alwaysApply: true
---
```

## Drift Detection

- Before writing, compute the source hash from the canonical encoding of `{ sortedModuleIds, versions, contentDigests, ignorePatterns }`.
- Read the existing target file (if any), extract the `source-hash` banner comment.
- If the existing file's hash differs from a hash recomputed over its current content body (excluding the banner), treat the file as hand-edited. Refuse to overwrite unless `--force` is supplied.
- Exit code 3 indicates drift refusal.

## Idempotency

When inputs are unchanged, `augx export` produces byte-identical output. Adapters are pure; the only externally-supplied value is the timestamp, which is replaced only when content has changed. Re-running with no changes is a no-op (no writes, exit 0).

## `--dry-run` Behavior

Prints, for each target:

```
[dry-run] claude-code -> CLAUDE.md (12_345 bytes, 7 modules)
[dry-run] cursor      -> .cursor/rules/augx.mdc (13_010 bytes, 7 modules)
```

No filesystem writes. Exit 0 if all targets are valid; non-zero on resolution errors.

## Ignore Patterns

`.augment/coordination.json.export.ignore` is an array of glob patterns evaluated relative to each module's root. Defaults to `["**/examples/**"]` to keep aggregated files within practical size budgets. Patterns use forward slashes regardless of host OS.

## Exit Codes

| Code | Meaning |
|---|---|
| 0 | Success (including no-op) |
| 1 | Usage error (missing/unknown `--target`, conflicting flags) |
| 2 | Resolution error (no linked modules, broken module) |
| 3 | Drift refusal (use `--force` to override) |
| 4 | I/O error (cannot write inside project root, permission denied) |

## Safety Rules

- Never writes outside the project root.
- Never follows symlinks out of `augment-extensions/` during resolution.
- All generated files use LF line endings regardless of host OS.
- All paths in generated content are forward-slash.

## Relationship to Other Surfaces

- Does NOT modify `.augment/` rules; the Augment Code surface remains unchanged.
- Does NOT call out to Beads or any MCP server; MCP is referenced only via the inlined wiring snippet.
- Does NOT touch `augment-extensions/modules/`; modules are read-only inputs.
