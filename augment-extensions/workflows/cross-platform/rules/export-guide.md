# `augx export` Day-to-Day Guide

## Purpose

`augx export` collects every linked Augx module's `rules/` content and writes it into the **single aggregated file** that one or more target tools expect.

## Grammar

```
augx export --target <claude-code|cursor|windsurf|copilot|all>
            [--output <path>]
            [--dry-run]
            [--force]
            [--verbose]
```

## Flags

| Flag | Default | Purpose |
|---|---|---|
| `--target` | (reads `.augment/coordination.json.export.targets`) | Pick the destination tool. Use `all` to write every target in one invocation. |
| `--output` | per-target conventional path | Override the destination path. Valid only with a single `--target`. |
| `--dry-run` | `false` | Print the resolved file list, projected byte counts, and module counts. No writes. |
| `--force` | `false` | Overwrite even if drift is detected (exit code 3). Use only after reviewing the diff. |
| `--verbose` | `false` | Print per-module sizes and mirror/symlink decisions. |

If `--target` is omitted **and** `.augment/coordination.json.export.targets` is empty or missing, `augx export` exits non-zero with a usage error.

## What Each Run Does

1. Reads `.augment/coordination.json` (and `.vscode/mcp.json`, `.augment/mcp/servers.json` for the MCP snippet check).
2. Loads every linked module, applies `export.ignore` glob patterns, computes a deterministic source hash.
3. Renders the aggregated content for each requested target.
4. Reads the existing target file (if any). If the banner's `source-hash` does not match the recomputed body hash, exits 3 (drift refusal). See `drift-resolution.md`.
5. If the new content is byte-identical to the existing content, exits 0 without writing (no-op).
6. Otherwise writes the new content with LF line endings.

## Output Anatomy

Every exported file begins with a banner:

```
<!-- GENERATED BY augx export - DO NOT EDIT -->
<!-- augx-version: <semver> -->
<!-- generated-at: <ISO-8601> -->
<!-- source-hash: <16-hex> -->
<!-- modules: <id>@<ver>, ... -->
```

Followed (when the Beads MCP server is declared) by the inlined MCP snippet, then per-module sections in **alphabetical order by module ID**, then a footer.

For Cursor, a YAML frontmatter precedes the banner:

```yaml
---
description: Augx aggregated rules
alwaysApply: true
---
```

## Typical Workflows

### First export in a project

```bash
augx link coding-standards/typescript
augx link workflows/openspec
augx export --target cursor
# writes .cursor/rules/augx.mdc with both modules
```

### Multi-target export

```bash
augx export --target all
```

Writes `CLAUDE.md`, `.cursor/rules/augx.mdc`, `.windsurfrules`, and `.github/copilot-instructions.md` in one pass. Each is computed independently; a failure on one target does not abort the others (per-target exit status is summarized at the end).

### Preview before writing

```bash
augx export --target all --dry-run
```

Output:

```
[dry-run] claude-code -> CLAUDE.md (12_345 bytes, 7 modules)
[dry-run] cursor      -> .cursor/rules/augx.mdc (13_010 bytes, 7 modules)
[dry-run] windsurf    -> .windsurfrules (12_120 bytes, 7 modules)
[dry-run] copilot     -> .github/copilot-instructions.md (12_405 bytes, 7 modules)
```

### Default-target setup

Add to `.augment/coordination.json`:

```json
{
  "export": {
    "targets": ["claude-code", "cursor"],
    "ignore": ["**/examples/**", "**/CHANGELOG.md"]
  }
}
```

After this, plain `augx export` writes both files. `export.targets` is the canonical place to record team conventions.

## Exit Codes

| Code | Meaning | Action |
|---|---|---|
| 0 | Success (including no-op) | Continue. |
| 1 | Usage error (missing/unknown `--target`, conflicting flags) | Fix the command. |
| 2 | Resolution error (no linked modules, broken module manifest) | Verify `augx list --linked`. |
| 3 | Drift refusal (target was hand-edited) | See `drift-resolution.md`. |
| 4 | I/O error (cannot write inside project root, permission denied) | Check filesystem permissions. |

## Idempotency Invariants

- **Same inputs -> byte-identical output.** The timestamp in the banner is only refreshed when the source hash changes; a no-op run leaves the file untouched.
- **Module order is canonicalized.** Modules are sorted by ID before rendering, so reordering `augx link` calls does not change the output.
- **Paths in content are forward-slash** regardless of host OS.
- **Line endings are LF** regardless of host OS.

## Safety Rules

- `augx export` never writes outside the project root.
- It never follows symlinks out of `augment-extensions/` while resolving sources.
- It does not call out to Beads, the MCP server, or the network.
- It never touches `.augment/` rules; the Augment Code surface stays as-is.

## What `augx export` Does NOT Do

- Does not embed live task state (`bd-` IDs, statuses, dependency lists).
- Does not modify `augment-extensions/modules/` content.
- Does not write per-module mirror files; that is `augx link --mirror`'s job.
- Does not delete files; only overwrites the target path or refuses (exit 3).
