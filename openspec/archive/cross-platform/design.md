# Design: cross-platform

## Overview

`augx` already resolves linked modules from `augment-extensions/modules/...` and exposes them to Augment Code via `.augment/`. This change adds a derivation layer that translates the same resolved module set into the native rule formats of four other AI coding tools (Claude Code, Cursor, Windsurf, GitHub Copilot), while keeping Beads task data out of static files and behind the Beads MCP server.

## Components

### 1. Module Resolver (existing, reused)

- Reads `.augment/coordination.json` to find linked modules.
- Returns an ordered list of `ResolvedModule { id, version, rootPath, rulesFiles[], examplesFiles[], readmePath }`.
- No changes required; export and mirror both consume this resolver output.

### 2. Per-Tool Adapter Layer (new)

One adapter per target tool. Each adapter implements a common interface:

```
interface ToolAdapter {
  id: 'claude-code' | 'cursor' | 'windsurf' | 'copilot'
  exportPaths(projectRoot): { single: string }            // single-file aggregator output path
  mirrorPath(projectRoot, moduleId): string                // per-module materialized path
  renderModule(module: ResolvedModule): string             // tool-specific section content
  renderHeader(meta: ExportMeta): string                   // banner + frontmatter
  renderFooter(meta: ExportMeta): string                   // attribution + canonical link
}
```

Adapters live under `cli/src/adapters/`. They are pure functions: given resolved modules + metadata, they return strings. All filesystem I/O happens in the runner.

### 3. Source Hash

- A stable hash of (sorted module IDs, versions, content digests, ignore patterns).
- Embedded in every generated file's header banner as `source-hash: <hex>`.
- On re-run, compare embedded hash with newly computed hash:
  - Equal -> no-op (byte-identical output guaranteed because adapters are deterministic).
  - Different + file unchanged since last write -> rewrite.
  - Different + file edited externally -> refuse unless `--force`.
- Algorithm: SHA-256 over a canonical JSON encoding of the inputs. Truncated to 16 hex chars for display.

### 4. Mirror Hook

- Runs at the end of `augx link <module>` when `--mirror <tools>` is provided or when `.augment/coordination.json.export.mirror === true`.
- Materialization strategy per file:
  1. Try `symlink(source, target)`.
  2. On `EPERM` / `EEXIST-with-different-target` / Windows-without-Developer-Mode, fall back to `copyFile(source, target)`.
  3. Record `{ moduleId, tool, sourcePath, targetPath, mode: 'symlink' | 'copy' }` in `.augment/coordination.json.mirrors[<moduleId>]`.
- Idempotency: before materializing, compare existing tracked entries to the desired set; create missing, update changed, delete stale.

### 5. Unlink Cleanup

- `augx unlink <module>` reads `.augment/coordination.json.mirrors[<moduleId>]`.
- For each tracked entry, deletes the target path (only if it still matches the tracked source / hash, to avoid clobbering hand-edited files).
- Removes the module's entry from `mirrors`.

## Data Flow

```
augment-extensions/modules/<m>/rules/*.md
                                  |
                                  v
                          Module Resolver
                                  |
              +-------------------+-------------------+
              |                                       |
              v                                       v
       Mirror Hook                            Aggregator (export)
       per-module symlink/copy                concat -> ToolAdapter
              |                                       |
              v                                       v
   .claude/rules/<m>/...                      CLAUDE.md
   .cursor/rules/augx-<m>.mdc                 .cursor/rules/augx.mdc
   .windsurf/rules/augx-<m>.md                .windsurfrules
   .github/instructions/augx-<m>...           .github/copilot-instructions.md
```

## Banner Format

```
<!-- GENERATED BY augx export - DO NOT EDIT -->
<!-- augx-version: <semver> -->
<!-- generated-at: <ISO-8601> -->
<!-- source-hash: <16-hex> -->
<!-- modules: <id>@<ver>, <id>@<ver>, ... -->
```

For Cursor `.mdc`, the banner is prepended as HTML comments after the YAML frontmatter.

## Cross-Platform Concerns

- **Line endings**: all generated files written with `\n` regardless of host OS; readers tolerant of CRLF.
- **Paths in content**: always forward-slash; convert with `path.posix.join` before serializing.
- **Symlinks on Windows**: `fs.symlinkSync` requires admin or Developer Mode. Detect `EPERM` and `ENOSYS`, fall back to copy without prompting. Record `mode: 'copy'` so updates re-copy rather than re-symlink.
- **Long paths on Windows**: prefer paths under the project root to stay within MAX_PATH; do not generate deeply nested mirror paths.

## MCP Wiring Snippet

Stored at `augment-extensions/workflows/mcp/templates/beads.md`. Inlined into exported files only when `.vscode/mcp.json` or `.augment/mcp/servers.json` declares a server with id `beads`. The snippet contains:
- Instruction to the consuming agent to query the Beads MCP server for live task state.
- A pointer to `augment-extensions/workflows/beads/` for command reference.
- No task IDs, no statuses, no live data.

## Error Handling

| Condition | Behavior |
|-----------|----------|
| Target file edited since last export | Exit non-zero unless `--force`; print diff hint |
| Symlink fails on Windows | Silent fallback to copy; log at `--verbose` only |
| Aggregate file exceeds 200 KB | Warn and continue; suggest tightening `ignore` |
| `--target` value unknown | Exit non-zero with list of valid targets |
| No linked modules | Exit zero, write empty banner-only file when explicit `--target` |

## Determinism Guarantees

- Module order in output follows alphabetical sort of `moduleId`.
- Adapters never read clock or random sources beyond the timestamp injected into the header.
- For tests, an `AUGX_FAKE_NOW` env var pins the timestamp to make byte-equality assertions practical.

## Backward Compatibility

- No existing command, flag, or file path is removed or renamed.
- Absence of `export` and `mirrors` in `.augment/coordination.json` is the legitimate default.
- `augx export` and `--mirror` are pure additions; pre-existing automation continues to work unmodified.
